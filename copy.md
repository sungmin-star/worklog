## 1. 서비스 개요

**L'Heure Bleue(작업일지)** 는 매일의 작업 흔적을 사진·짧은 글·태그로 남기는 1인용 일지 앱입니다.
설치형 웹앱(PWA)으로 폰·PC 어디서나 동작하며, 로그인하면 기기 간 자동 동기화됩니다.

| 성격 | 내용 |
|---|---|
| 📝 기록 중심 | 날짜·시간·분류·사진·태그로 빠르게 기록 |
| 🔗 연결 사고 | `[[위키링크]]`·태그로 글을 잇고 그래프로 시각화 |
| ✦ AI 비서 '자비스' | 기록을 근거로 요약·회고·할일 안내 |
| 📱 오프라인 PWA | 설치·오프라인 사용·자동 업데이트 |

## 3. 기술 구성

| 영역 | 구성 |
|---|---|
| 프런트엔드 | 단일 `worklog.html` 파일 (HTML + CSS + Vanilla JS, 빌드 도구 없음) |
| 오프라인 | `sw.js` 서비스 워커 — 앱 셸 캐시, 버전 올리면 자동 갱신 |
| 설치(PWA) | `manifest.json` + 아이콘(192/512/180) |
| 백엔드 | Supabase (인증 · DB · 이미지 Storage · Edge Function) |
| AI | Supabase Edge Function `jarvis` → Anthropic Claude API |
| 호스팅 | GitHub Pages (정적) |

> 빌드/번들러가 없어 `worklog.html`을 그대로 수정하면 됩니다. 외부 의존성은 CDN(Supabase JS SDK)만 사용합니다.

## 4. 홈페이지 전체 흐름

사용자가 앱을 켜고 쓰는 순서대로의 화면 흐름입니다.

### 4-1. 진입 · 인증
1. **로그인 / 회원가입** — 이메일·비밀번호 (가입 시 이름·생년월일). 비밀번호는 **8자 이상 + 영문 대·소문자·숫자 필수**.
2. **자동 로그인** — 세션 영구 저장(특히 iOS 설치형 PWA), 마지막 이메일 기억.
3. **비밀번호 재설정** — 메일 링크로 새 비밀번호 설정.
4. **화면 잠금**(선택) — PIN + 생체인증(WebAuthn). 잠금 해제 후 데이터 로드.

### 4-2. 메인 화면 (타임라인)
- **헤더** — 로고 / 라이트·다크 테마 전환 / 메뉴(⋮)
- **검색** — 본문 검색 + 연산자(`tag:태그`, `-제외`, `"정확히"`)
- **새 기록** 버튼
- **목록 / 카드 보기 토글** — 기본은 *한 줄 목록*(한 화면에 12~15개), 탭하면 전체보기. 카드 모드로도 전환 가능. 선택은 기기에 저장.
- **월별 · 태그 필터** 바
- **기록 목록** — 분류 색점 · 날짜 · 시간 · 첫 줄 · 분류 뱃지 · ☆고정 버튼

### 4-3. 작성 · 수정 (에디터)
- 날짜 **자동 인식**(첫 줄에 날짜 쓰면 반영) · 시간 · 분류(카테고리) 선택
- 사진 첨부(드래그/선택, 자동 압축) · `#태그` · `[[위키링크]]`
- 🎤 **음성 입력**(한국어 실시간 받아쓰기)
- 💾 **작성 중 임시저장** — 앱 전환·새로고침·크래시에도 글이 안 날아가고 "이어서 쓰기"로 복구

### 4-4. 읽기 · 모아보기
- **전체보기** — 메모를 탭하면 읽기 전용 전체보기. 안에서 ☆고정 / ✏️수정 / 🗑삭제
- **★ 자주 보는 메모** — 화면 왼쪽 아래 버튼 → 고정한 메모만 빠르게. PC는 "새 창"으로 띄워 엑셀 옆에 두고 참조
- **📖 업무 노트** — 분류 선택 → 접기/펼치기 + 검색되는 읽기 전용 매뉴얼 뷰

### 4-5. 부가 화면
- **📅 캘린더** — 일정 시각 표시, 날짜별 기록 열람·작성
- **🕸 그래프** — 분류 / `[[위키링크]]` 기준 관계 맵
- **🖼 사진 갤러리** — 전체 사진 모아보기
- **🗂 프로젝트 · 할일** — 목표 + 체크리스트, 진행률
- **✦ 자비스** — 화면 오른쪽 아래 FAB, 기록 근거 요약·질의응답

### 4-6. 메뉴
오늘 일지 · 📖 업무 노트 · 🗂 프로젝트 · ✦ AI 요약 · 백업 내려받기/가져오기 · 📄 인수인계 자료(PDF) · 작업 일지에 대하여 · 🗑 휴지통 · 🔒 화면 잠금 · 로그아웃

## 5. 데이터 · 저장소

로그인 여부에 따라 저장소가 달라집니다.

| 상태 | 저장 위치 |
|---|---|
| 비로그인 | 브라우저 `localStorage` (그 기기 전용) |
| 로그인 | Supabase → 기기 간 동기화. 이미지는 비공개 Storage 버킷 `worklog-images`에 업로드, 화면엔 서명 URL(유효 1일) |
| 오프라인 | 변경을 로컬 큐에 쌓아 두었다가 복구 시 재전송 |

**주요 데이터 단위** — 기록(entry), 휴지통(trash, 30일 복원), 프로젝트(project: 목표·할일), 분류(category).
모든 데이터는 "백업 내려받기(JSON)"로 언제든 이전 가능합니다.

## 6. Supabase 백엔드

| 항목 | 내용 |
|---|---|
| 프로젝트 | 리전 ap-northeast-2 · sungmin-star 본인 계정 |
| 인증 | 이메일/비밀번호 (이름·생년월일 메타데이터) · **8자+영문 대·소문자·숫자 필수** |
| 보안 | 모든 테이블 **RLS 활성화**(사용자별 데이터 격리) |
| 이미지 Storage | 비공개 버킷 `worklog-images` · 서명 URL로 표시 |
| Edge Function | `jarvis` (ACTIVE) — 질문 + 기록을 받아 Claude API 호출 후 답변 |
| AI 모델 | 기본 sonnet (haiku/opus 전환 가능) · 입력 컨텍스트 약 14K자 |

> **AI 백엔드 활성화 스위치** · `jarvis` 함수는 배포·연결돼 있고, 현재 `ANTHROPIC_API_KEY` 시크릿만 비어 있어 "복사 폴백" 모드입니다. 이 키만 설정하면 실제 AI 응답으로 작동합니다.
>
> **유료 제약** · 유출 비밀번호 대조(HaveIBeenPwned)는 Supabase Pro 전용이라 무료 플랜에선 못 켭니다. 대신 위 비밀번호 정책(8자+종류 강제)으로 보완했습니다.
>
> 실제 API 키·서비스 키·비밀번호는 이 문서/저장소에 넣지 않습니다.

## 7. 배포 절차 (중요)

1. 로컬 `worklog.html` 수정
2. **`sw.js`의 캐시 버전을 반드시 +1** (예: `worklog-v39 → v40`) — 이용자 앱 자동 갱신의 핵심
3. 로컬 git 커밋 (기록용)
4. 게시 저장소에 업로드: `worklog.html → index.html`, `sw.js → sw.js` (sungmin-star PAT로 GitHub Contents API에 디스크 파일 그대로 byte-exact PUT)
5. 업로드 후 blob SHA를 로컬 `git hash-object`와 대조해 무결성 검증
6. 1분 내 GitHub Pages 반영 → 폰에서 자동 업데이트
7. 작업 종료 시 `handoff-src.html` 갱신 → Chrome으로 `HANDOFF.pdf` 재생성 → 같이 업로드

> ⚠️ 게시 저장소는 **public**입니다. 어떤 비밀 키·비밀번호도 `index.html`/문서에 넣지 마세요. (공개용 publishable anon 키만 포함)
