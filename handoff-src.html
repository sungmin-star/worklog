서비스 개요

L'Heure Bleue(작업일지)는 매일의 작업 흔적을 사진·짧은 글·태그로 남기는 1인용 일지 앱입니다. 설치형 웹앱(PWA)으로 폰·PC 어디서나 동작하며, 로그인하면 기기 간 자동 동기화됩니다.

📝 기록 중심날짜·시간·분류·사진·태그로 빠르게 기록
🔗 연결 사고[[위키링크]]·태그로 글을 잇고 그래프로 시각화
✦ AI 비서 '자비스'기록을 근거로 요약·회고·할일 안내
📱 오프라인 PWA설치·오프라인 사용·자동 업데이트

기술 구성

프런트엔드단일 worklog.html 파일 (HTML+CSS+Vanilla JS, 빌드 도구 없음)
오프라인sw.js 서비스 워커 — 앱 셸 캐시, 버전 올리면 자동 갱신
설치(PWA)manifest.json + 아이콘(192/512/180)
백엔드Supabase (인증 · DB · 이미지 Storage · Edge Function)
AISupabase Edge Function jarvis → Anthropic Claude API 호출
호스팅GitHub Pages (정적)

※ 빌드/번들러가 없어 worklog.html을 그대로 수정하면 됩니다. 의존성은 CDN(Supabase JS SDK)만 사용합니다.

주요 기능

타임라인 기록날짜·작성시각·분류·사진·#태그·[[링크]]·⭐즐겨찾기
👁 메모 전체보기NEW메모를 누르면 읽기 전용 전체보기, 수정은 따로 버튼. v33
🎤 음성 입력본문을 말로 받아쓰기 (한국어, 실시간). v34에서 중지(취소) 안정화
📅 캘린더일정 시각 표시, 날짜별 기록 열람·작성
🕸 그래프분류·[[위키링크]] 기준 관계 맵 시각화
🖼 사진 갤러리전체 사진 모아보기
🗂 프로젝트·할일목표 + 체크리스트, 진행률 관리
✦ AI 요약·자비스기록 근거 요약·회고·질의응답·할일 안내
🔒 화면 잠금PIN + 생체인증(WebAuthn)
☁ 클라우드 동기화로그인 시 기기 간 동기화(이미지 포함)
💾 백업JSON 내보내기 / 가져오기
👁 메모 전체보기 — 타임라인 카드 본문이나 캘린더·그래프·자비스에서 메모를 누르면, 수정창이 아니라 내용을 한눈에 읽는 전체보기가 열립니다. 수정은 그 안의 ✏️ 수정 버튼(또는 카드의 연필 아이콘)으로만 들어가도록 분리했습니다.
🎤 음성 입력 사용법 — 새 기록/수정 화면의 본문 라벨 옆 🎤 음성 버튼을 누르면 받아쓰기가 시작됩니다. 말하면 실시간으로 본문에 입력되고, 다시 누르면 멈춥니다. 마이크 권한과 HTTPS가 필요하며, 미지원 브라우저에서는 버튼이 자동으로 숨겨집니다. (v34: 모바일에서 중지 버튼이 듣지 않던 문제를 고쳐, 누르는 즉시 멈춥니다.)
5

데이터 · 저장소

로그인 여부에 따라 저장소가 달라집니다.

비로그인브라우저 localStorage에만 저장 (그 기기 전용)
로그인Supabase를 저장소로 사용 → 기기 간 동기화. 이미지는 비공개 Storage 버킷 worklog-images에 업로드되고 화면엔 서명 URL로 표시
오프라인 큐네트워크가 끊기면 변경을 로컬 큐에 쌓아 두었다가 복구 시 재전송

주요 데이터 단위 — 기록(entry), 휴지통(trash), 프로젝트(project: 목표·할일), 분류(category). 모든 데이터는 “백업 내려받기(JSON)”로 언제든 이전 가능합니다.

6

Supabase 백엔드 구성

프로젝트리전 ap-northeast-2 · 본인(sungmin-star) Supabase 계정
인증이메일/비밀번호 (가입 시 이름·생년월일 메타데이터)
이미지 Storage비공개 버킷 worklog-images · 서명 URL(유효 1일)로 표시
Edge Functionjarvis (ACTIVE) — 질문 + 기록을 받아 Claude API 호출 후 답변 반환
AI 모델기본 sonnet (haiku/opus 전환 가능) · 입력 컨텍스트 약 14K자 제한
AI 백엔드 활성화 스위치 · jarvis 함수는 이미 배포되어 있고 프런트도 연결돼 있습니다. 현재는 함수의 ANTHROPIC_API_KEY 시크릿이 비어 있어 “복사 폴백” 모드로 동작합니다. 이 키만 설정하면 ✦AI 요약·자비스 자유질문이 즉시 실제 AI 응답으로 작동합니다.

※ 보안: 실제 API 키·서비스 키·계정 비밀번호는 이 문서에 포함하지 않습니다. 키 값은 Supabase 대시보드(Edge Function Secrets)와 로컬 설정 파일에서 관리합니다.

배포 절차 (중요)

  1. 로컬 worklog.html 수정
  2. sw.js의 캐시 버전을 반드시 +1 (예: worklog-v29 → v30) — 이용자 앱이 자동 갱신되는 핵심
  3. 로컬 git 커밋 (기록용)
  4. 게시 저장소에 업로드: worklog.html → index.html, sw.js → sw.js
    sungmin-star 권한으로 GitHub Contents API 또는 git push. 224KB 본문은 디스크 파일을 그대로(바이트 정확) 올릴 것
  5. 업로드 후 blob SHA를 로컬 git hash-object와 대조해 무결성 검증
  6. 1분 내 GitHub Pages 반영 → 폰에서 자동 업데이트
주의 · 게시 저장소는 public입니다. 어떤 비밀 키·비밀번호도 index.html/문서에 넣지 마세요. (공개용 publishable anon 키만 포함)
